1) WEEKLY COACHING + ADAPTIVE GOAL ENGINE
What to build
- A weekly coaching module that analyzes the last 7-28 days and proposes:
  - Keep, increase, or decrease deficit/surplus
  - Suggested macro target adjustments
  - Suggested meal timing patterns
- Add transparent explanation text: “Recommendation based on adherence, weight trend, and energy balance estimate.”

Why it matters
- Most users fail from static targets that become unrealistic over time.
- Existing burn/goal logic is strong; adaptive coaching turns raw data into decisions.

User impact
- Better consistency and lower drop-off.
- Reduced frustration when progress stalls.

Implementation notes
- Reuse existing trend data and calibration state.
- Build a recommendation engine with rule-based first version:
  - If adherence high + no expected progress -> adjust targets conservatively.
  - If adherence low -> reduce aggressiveness and prioritize consistency.
- Later version: optional ML personalization.

Risks/complexity
- Medium: recommendations must avoid unsafe aggressiveness.
- Requires careful guardrails and user education copy.

Success metrics
- 4-week retention increase.
- Higher goal adherence rate.
- More weeks with sustained logging.

2) STREAKS, ADHERENCE SCORING, AND BEHAVIORAL NUDGES
What to build
- Daily adherence score (0-100) based on:
  - Logging completeness
  - Calorie proximity to target range
  - Macro target alignment (if tracked)
- Streak system with flexible rules (e.g., “good enough day” tolerance).
- Nudges triggered by behavior (e.g., missed lunch logging pattern).

Why it matters
- Progress is usually driven by behavior consistency, not perfect days.
- Current app has strong tracking but limited motivational feedback loops.

User impact
- Makes progress visible even when scale changes are slow.
- Reduces all-or-nothing mindset.

Implementation notes
- Add `adherenceScore` per day in history model.
- Create nudge event engine with hard caps (avoid notification fatigue).
- Provide settings to mute categories of nudges.

Risks/complexity
- Low-medium: risk of over-notifying users.

Success metrics
- Increase average logged days per month.
- Improve week-over-week adherence score.

3) MEAL PLANNING + PLAN VS ACTUAL TRACKING
What to build
- Users can pre-plan meals for next day/week.
- Today screen compares planned vs consumed calories/macros.
- “Swap suggestions” when plan deviates.

Why it matters
- Pre-commitment reduces impulsive eating and logging gaps.
- Bridges tracking with forward-looking control.

User impact
- More predictable progress and easier grocery prep.

Implementation notes
- Add `PlannedMealEntry` model parallel to `MealEntry`.
- Integrate quick-add, saved meals, and menu imports into planner.
- Show “remaining planned macros” card in Today tab.

Risks/complexity
- Medium: UI complexity and calendar planning UX.

Success metrics
- % of users creating plans.
- Lower day-to-day calorie variance.

----------------------------------------------------------------------

4) RECIPE BUILDER + SAVED MEALS + LEFTOVER PORTIONS
What to build
- Recipe creation with ingredients from USDA/barcode/manual items.
- Auto compute per-serving nutrition by weight or portion count.
- Save entire meals and one-tap re-log.
- Leftover mode: divide into N portions or remaining grams.

Why it matters
- Home-cooked meals are hardest to log accurately.
- Frequent meals should be near-zero friction to re-enter.

User impact
- Massive reduction in logging time.
- Better nutrition accuracy for repeated home meals.

Implementation notes
- New models: `Recipe`, `RecipeIngredient`, `PreparedMealInstance`.
- Add conversion utility for edible portion and serving normalization.
- Optional photo attachment for recognition context.

Risks/complexity
- Medium-high: unit conversion and UX complexity.

Success metrics
- Increase repeat-log usage.
- Decrease average time-to-log meal.

5) MACRO PERIODIZATION (TRAINING VS REST DAY GOALS)
What to build
- Goal templates by day type:
  - Strength training day
  - Endurance day
  - Rest day
- Auto switch based on workout schedule or detected workout.

Why it matters

- Existing burn model can support smarter macro distribution.

User impact

Implementation notes

- Map workouts from HealthKit to profile selection.
- Include manual override per day.

Risks/complexity

Success metrics
- Higher macro target hit rate on training days.
- Better perceived usefulness among active users.

6) CONFIDENCE / UNCERTAINTY INDICATORS FOR AI & IMPORTED DATA
What to build
- Show confidence score and uncertainty range on AI/photo/menu parsed nutrition.
- Highlight fields likely inaccurate and suggest quick edits.

Why it matters
- Users trust systems more when uncertainty is explicit.
- Prevents silent error accumulation in calorie estimates.

User impact
- Better decision confidence and fewer bad logs.

Implementation notes
- Backend returns confidence metadata per item and nutrient.
- UI badges: High/Medium/Low confidence + “Review suggested” indicator.
- Track correction rates to improve prompts/parsing.

Risks/complexity
- Medium: requires schema changes in AI endpoints.

Success metrics
- Reduced post-add edits caused by incorrect AI outputs.
- Better user trust ratings.

8) NUTRITION QUALITY SCORE (MICRO + FIBER + SODIUM BALANCE)
What to build
- A quality score that complements calories/macros.
- Include weighted components: protein adequacy, fiber intake, added sugar limit, sodium range, micronutrient coverage.

Why it matters
- Users can hit calories while still eating poor-quality nutrition.
- Adds health perspective beyond energy balance.

User impact
- Better long-term health outcomes, not just weight changes.

Implementation notes
- Use currently tracked nutrients where available; gracefully degrade when data missing.
- Show both daily and weekly score trend.

Risks/complexity
- Medium: avoid pseudo-scientific scoring; keep transparent formulas.

Success metrics
- Increased fiber/protein adherence.
- Reduction in high-sodium or high-added-sugar days.

Risks/complexity

Success metrics
- Improved AI portion accuracy against validation set.
